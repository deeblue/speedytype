# 第二輪真人語音讀稿腳本

請固定錄音環境：同一個房間、同一支麥克風、盡量固定嘴巴到麥克風的距離，並讓整段錄音的環境噪音保持一致。不要像上一輪那樣最後一句才突然變安靜。

1. **短句** [停頓提示：自然念完即可] 呃，請 QA 今天補上 BIOS 測試結果。
2. **短句** [停頓提示：USB 後停一下] 那個，USB 相容性先給 BJ 團隊確認。
3. **短句** [停頓提示：API 前後自然停頓] 請把 API 規格同步給 TPE 團隊。
4. **中句** [停頓提示：Firmware 後稍停] Firmware 版本更新後，Thunderbolt 的回歸測試要再跑一次。
5. **中句** [停頓提示：NPI 前稍停] 我們這週的 NPI 進度先讓 PD 團隊看過，再交給 QA 整理。
6. **中句** [停頓提示：BJ 團隊後停一下] 請 BJ 團隊先看 BIOS 設定，再把異常回報給 Firmware owner。
7. **中句** [停頓提示：API 後停一下] 就是說，API 測試失敗的案例，先不要關單，等 TPE 團隊確認根因。
8. **中句** [停頓提示：USB 後稍停] 然後 USB 和 Thunderbolt 的問題單，要在下次 NPI 會議前整理完。
9. **中句** [停頓提示：自我修正處自然停頓] 我本來想請 BJ 團隊處理，啊不對，這題應該先給 TPE 團隊確認。
10. **中句** [停頓提示：自我修正處自然停頓] 明天先寄 API 文件，不對，應該先寄 NPI checklist 給 QA。
11. **長句** [停頓提示：每個步驟中間稍停] 今天要完成三件事，第一，更新 BIOS 和 Firmware 的版本說明，第二，把 API 測試結果寄給 QA 和 BJ 團隊，第三，確認 USB 跟 Thunderbolt 的相容性風險。
12. **長句** [停頓提示：每個團隊名稱後稍停] 這週專案同步時，TPE 團隊要先講 BIOS 狀態，BJ 團隊補充 Firmware 風險，QA 再說明 NPI 測試進度。
13. **長句** [停頓提示：API 與 NPI 之間停一下] 如果今天會議時間不夠，就先講 API 變更、Firmware 版本差異，還有下週 NPI 需要追的項目。
14. **長句** [停頓提示：自我修正處自然停頓] 我們原本打算先測 USB，再測 Thunderbolt，啊不對，應該先把 BIOS 更新完再一起驗證。
15. **長句** [停頓提示：條列感自然念出來] 請你記下三個重點，第一個是 API 規格不要再改，第二個是 QA 今天要補 BIOS log，第三個是 TPE 團隊和 BJ 團隊都要回報 Firmware 狀態。
16. **長句** [停頓提示：最後一串專有名詞放慢一點] 如果辦公室很吵，還是請先用 SpeedyType 記錄問題，內容至少要包含 BIOS、Firmware、NPI、API、USB 和 Thunderbolt，之後再人工確認一次。
